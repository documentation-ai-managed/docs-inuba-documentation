## Quick Upgrade Guide

Follow these steps to upgrade to the latest version of Inuba Documentation.

<Steps>
  <Step title="Backup Your Site" icon="database">
    Export your current documentation site.

    ```bash
    npx inuba-export --site=my-docs
    ```
  </Step>
  <Step title="Update Package" icon="package">
    Install the latest version using your package manager.
  </Step>
  <Step title="Rebuild and Deploy" icon="rocket">
    <Tabs>
      <Tab title="Vercel" icon="cloud">
        ```bash
        vercel --prod
        ```
      </Tab>
      <Tab title="Netlify" icon="monitor">
        ```bash
        netlify deploy --prod --dir=dist
        ```
      </Tab>
    </Tabs>
  </Step>
</Steps>

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added `<Tabs>` and `<Steps>` components for better tutorials and platform-specific guides
- Introduced `<Columns>` layout for feature grids and navigation cards
- Enhanced search with autocomplete and category filters

## Improvements

- Optimized MDX parsing speed by 30% for larger sites
- Improved mobile responsiveness for all components

## Bug Fixes

- Fixed parsing errors with nested code blocks
- Resolved icon rendering issues in dark mode

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "bugfix"]}>

## New Features

- Released `<Update>` component for streamlined changelogs
- Added `<Callout>` with multiple kinds (`info`, `tip`, `success`, `alert`, `danger`)
- Support for custom brand colors like `#3B82F6`

## Bug Fixes

- Corrected frontmatter YAML parsing for descriptions with special characters
- Fixed heading hierarchy enforcement (no more H1 in body content)

<Update label="2024-08-10" description="v1.0.0" tags={["breaking", "feature"]}>

## New Features

- Initial release with core MDX components: `<Card>`, `<Request>`, `<Response>`
- Full support for Mermaid diagrams as code blocks
- Integrated Lucide icons for all visual elements

## Breaking Changes

- Removed legacy `<Mermaid>` component; use code blocks only
- Enforced strict escaping for `{`, `}`, `<`, `>` in prose

## Improvements

- Added verification checklist for zero parsing errors

</Update>
